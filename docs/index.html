Перший вхід в систему

Після установки АС Журналу, він вимкнений. Авторизуватись може тільки Адміністратор з паролем: admin

Після авторизації перейдіть до меню "Адміністрування" - "Глобальні налаштування"

Глобальні Налаштування

Натисніть кнопку "Редагувати настройки", щоб перейти до... ви не повірете... до редагування настройок.

Опис всіх налаштувань стр.1

1. Доступ до АС Журнал - Якщо доступ вимкнений, авторизуватись може тільки користувач з правами адміністратора.

2. Назва сайту - назва вашого УДКСУ, або організації.

3. Код району - використовується для генерації імені файлів та теми листа.

4. Додаткова директорія для архівів - Папка, на яку не діє правило ліміту резервних файлів БД, може бути мережевою.

5. Ліміт копій БД (шт.) - К-ть копій БД, які зберігаються в папці inc/backups/

Опис всіх налаштувань стр.2

6. Архіватор ZIP - Авто-перевірка підключення розширення ZIP в PHP.ini extension=php_zip.dll. Якщо присутне розширення, архівування БД та вхідної кореспонденції по IMAP буде в заархівованому виді zip.

7. Таймаут для авторизації (сек.) - Після закінчення терміну авторизації, користувачу потрібно авторизуватись ще раз. 0 - для відключення.

8. К-ть номерів на сторінку - К-ть записів що будуть показуватись користувачу на одній сторінці, якщо він не вказав іншу к-ть.

9. Макс. к-ть записів на одну сторінку - Максимально допустиме значення, яке буде переписане користувачеві в разі збільшення.

10. Вибір прізвища при авторизації - Якщо дозволено, користувачеві не потрібно вводити свій логін при авторизації, просто обрати своє прізвище із списку.

Опис всіх налаштувань стр.3

11. Рік запуску АС Журнал - Рік, з якого почав працювати АС Журнал. Користувачі не матимуть змоги перейти до меньшо року ніж цей.

12. Дозволені типи файлів - Розширення файлів, які дозволені до зберігання.

13. Максимальний розмір файлу (МБ) - Файли більшого розміру не будуть збережені.

14. Періодичність копіювання БД (сек.) - Резервне копіювання БД. Рекомендовано 3600 (раз на годину).

15. Періодичність Відправки БД на e-mail (сек.) - Періодичність відправлення свіжої копії БД на пошту адміністратора. Рекомендовано 36000 (раз на десять годин).

Збереження налаштувань

Для збереження налаштувань, натисніть "Відправити"

Якщо всі записи вірні, отримаєте повідомлення про успішне зберігання.

Зачейкайте переадресації (не обовязково)

PhpInfo + Створення архіву + Відправка архіву

Для перегляду інформації по PHP натисніть "PhpInfo"

Щоб не чекати таймауту операції по архівуванню та відправці копії Бази Даних, можете ініціалізувати вручну шляхом натискання "Створити архів" чи "Створити архів БД та відправити на пошту".

Управління довідниками

Якщо у користувача є права модератора йому доступне меню "Адміністрування" / "Довідники"

Управління довідниками - Структура

Є можливість додавати нові записи, редагувати та вилучати записи. Будьте уважні при редагуванні структури, краще вилучіть не вірні дані, а введіть вірні через форму додавання. Редагування приведе до змін по всій базі даних.

Управління довідниками - Номенклатура

Є можливість додавати нові записи, редагувати та вилучати записи. Будьте уважні при редагуванні номенклатури, краще вилучіть не вірні дані, а введіть вірні через форму додавання. Редагування приведе до змін по всій базі даних.

Управління Користувачами

Якщо у користувача є права на управління користувачами, йому доступне меню "Адміністрування" / "Управління користувачами"

Дії з користувачами

В даному розділі, можна додавати нових, редагувати, вилучати та активувати/деактивувати користувачів

Введення нового користувача стр.1

1. Логін* - Рекомендуеться продублювати логін домена (AD).

2. Введіть новий пароль* - Майбутній пароль користувача.

3. Повторіть новий пароль* - Повторіть пароль користувача.

4. Прізвище І.П.* - Ім'я користувача в системі, рекомендується по масці: Прізвище І.П.


* - Можна використувувати будь-які символи, латиниці, кирилиці, знаки, цифри, єрогліфи, спецсимволи... взагалі будь-які.

Введення нового користувача стр.2

5. Структура* - Оберіть стуктуру для користувача.


* - Для вибору кількох - зажміть CTRL на клавіатурі та клікайте мишкою, повторний клік зніме виділення.

** - Перед додаванням користувача, потрібно заповнити довідники Структури та Номенклатури, якщо дане поле буде порожнім, додавання користувача не відбудеться.

Введення нового користувача стр.3

6. Присвоєний IP* - Дане поле буде звірятись, з даними "Авторизація по IP" та "Авторизація з іншого IP".

7. Внутрішня ел. адреса* - E-mail в організації.

8. Зовнішня ел. адреса* - Особистий E-mail.

9. № IP телефону* - Прямий номер телефону.

10. № Стаціонарного тел.* - Міський номер телефону.

11. № Моб. телефону* - Мобільний номер телефону.


* - Не обов'язкове до заповнення.

Введення нового користувача стр.4

6. Авторизація з іншого IP - Дозволити користувачу авторизуватись з іншої IP-адреси, ніж вказана в полі "Присвоєний IP".

7. Авторизація по IP - Якщо користувач відкриває АС Журнал з IP-адреси вказаної в полі "Присвоєний IP", відбуває автоматичне розпізнавання користувача.

8. Мова на сайті - мовний файл, який знаходиться в папці inc/lang/*.php, де * - це ініціали мови.

9. Права користувача - Право на зміну особистого паролю.

Введення нового користувача стр.5

6. Зміна глобальних налаштувань - Дозволити користувачу доступ до меню "Глобальні налаштування".

7. Перегляд журналу роботи - Дозволити користувачу доступ до меню "Перегляд журналу роботи". Технічна інформація про роботу АС Журнал.

8. Управління користувачами - Дозволити користувачу доступ до меню "Управління користувачами".

9. Управління доступом - Дозволити користувачу доступ до меню "Управління доступом". Можливість блокування/розблокування IP адрес.

10. Права модератора - Дає доступ до: "Довідників", можливіть редагувати всі введені номери в журналах (при цьому доступна вся номенклатура та управління файлами), вилучати останній введений номер, використовувати модераторскі команди в чаті.

Введення нового користувача стр.6

11. Активувати - Дозволити працювати в системі, якщо користувач де-активований - він не може авторизуватись.

12. Настройки приватності* - Оберіть, які права в журналах буде мати користувач. Якщо відсутні права "Бачити всю..." - користувачу будуть доступні тільки його записи.

Для збереження даних натисніть "Відправити". Якщо відсутні помилки, зміни будуть збережені.


* - Для вибору кількох - зажміть CTRL на клавіатурі та клікайте мишкою, повторний клік зніме виділення.

Перегляд журналу роботи

Якщо у користувач є права на "Перегляд журналу роботи" йому доступне відповідне меню в "Адміністрування".


В даному меню користувач може ознайомитись з технічною інформацією, яка наповнюється автоматично. Вилучення даних не можливе. Дані не потрапляють в резервну копію.

Управління доступом

Якщо у користувач є права на "Управління доступом" йому доступне відповідне меню в "Адміністрування".


В даному меню користувач може заблокувати доступ по IP до журналу, або відновити його вилучивши блокування.
